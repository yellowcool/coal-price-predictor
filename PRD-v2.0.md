# "煤探长 AI"MVP 一期产品需求文档（PRD）

**文档版本：** V2.0 (工程落地最终版)  
**面向对象：** 架构师、后端研发、算法工程师、爬虫工程师、前端研发  
**核心原则：** 控成本、重证据、保胜率、严控商业越界风险。

---

## 一、产品概述与核心预测逻辑

### 1. 核心业务目标

构建一个基于"大模型 Agent + 传统量化工具"的混合预测 SaaS 系统。系统通过异步定时任务运行，输出特定炼焦煤标杆 SKU 的**旬度（10 天）现货价格波动趋势**，以及**中长期（未来 3-6 旬）**的宏观定调。

- **不预测绝对价格**，仅预测涨跌方向及百分比涨跌幅区间（Delta %），前端再换算为绝对数值。
- **拒绝大模型无端脑补**，必须生成基于**可审计证据链（Evidence-driven）**的结构化推演战报。
- **坚决不提供配煤比例修改建议**（系统 Prompt 层需严格写死，规避生产事故连带责任）。

### 2. 预测目标与时间窗的数学定义（算法对齐标准）

- **基准价格锚点（P0）：** `P0 = mean(price[T-9 ... T])`（即 T 日向前推 10 天的现货均价）。
- **未来一旬预测目标（Y1）：** `Y1 = mean(price[T+1 ... T+10])`（即未来 10 天的预测均价）。
- **算法底层输出单位：** 统一输出百分比变化率区间（例如 Delta_pct_range: [+1.5%, +2.5%]）。前端渲染时，通过公式（`P0 × 百分比`）换算为"元/吨"展示给客户。

### 3. 一期（MVP）标杆 SKU 池

底层架构必须支持 SKU 的动态横向扩展，但一期仅需跑通以下 3 个核心基准 SKU 的预测闭环：

| SKU ID | 名称 | 描述 |
|--------|------|------|
| SKU_01 | 山西安泽低硫主焦煤 | 高利润弹性骨架煤 |
| SKU_02 | 山西吕梁柳林高硫主焦煤 | 主流抗跌配煤 |
| SKU_03 | 内蒙古甘其毛都口岸蒙 5#原煤 | 进口冲击风向标 |

## 二、系统总体架构与技术流转

系统采取经典的三层架构，核心聚焦于数据的高效流转与 AI 的无缝组装。

### 1. 数据获取层（Data Pipeline）

- **结构化数据源：** 采买 Mysteel、汾渭能源等 API，通过定时调度系统（Airflow/Celery）拉取，落盘至时序数据库。
- **非结构化数据源：** 自研高频爬虫，抓取政务公告、天气预警，清洗去重后落盘至文档数据库，并进行向量化处理存入向量数据库，供 RAG 检索。

### 2. AI 逻辑中枢层（Agentic Workflow）

- **中枢大脑（Chief LLM）：** 采用具备长文本逻辑推理能力的基座模型。
- **外挂技能包（Tools/Skills）：**
  - Tool_A_Quant: 时序预测模型，输出基线涨跌幅数值与特征重要性打分。
  - Tool_B_NLP: 情绪解析规则引擎与 NLP 提取，输出事件极性、强度打分与证据片段。

### 3. 业务展示层（B2B SaaS）

- 响应式 Web 端看板。核心逻辑为"权限鉴权驱动渲染"，用户只能看到己方付费购买的 SKU 预测数据。

## 三、数据工程需求

### 3.1 结构化特征数据（API 定时任务）

**核心特征池：** 247 家钢厂日均铁水产量（周）、全国高炉开工率（周）、110 家洗煤厂开工率（周）、甘其毛都口岸日均通关车数（日）、247 家钢厂炼焦煤可用天数（周）、三大核心 SKU 历史出厂/通关价（日）。

**⚠️ 数据对齐硬性规范（周频转日频）：** 所有周频发布的数据，必须采用**"前向填充（Forward Fill / ffill）"**策略。

### 3.2 非结构化情报数据（爬虫开发要求）

爬虫工程师针对国家及晋蒙陕安监局官网、中国气象局 API 开发高频抓取脚本（每 4 小时一次）。

**清洗与入库规范：**
- `event_region`：发生地（安泽/柳林/甘其毛都等）
- `start_time / expected_duration_hours`：事件持续时间
- `dedupe_key`：去重键（标题 hash+ 来源 + 日期）
- `raw_text_snippet`：用于后续作为审计证据链的原文片段

## 四、AI 算法与工作流引擎需求（核心逻辑）

### Step 1: 运行量化基线模型（Tool_A_Quant）

- **触发时机：** 每日固定时间（如凌晨 01:00）跑批。
- **输出约定：** JSON 格式。必须包含量化得分，LLM 无权自造权重。

### Step 2: 运行 NLP 情报提取（Tool_B_NLP）

- **输入：** 当日最新抓取且去重的非结构化文本。
- **输出约定：** JSON 格式。结合预设规则库输出事件强度。

### Step 3: 主 Agent 融合与战报生成（Chief LLM）

- **任务：** 将 Step1 和 Step2 的 JSON 输入主 Agent。LLM 仅负责将两者的打分进行数学**归一化（Normalization）**凑成 100% 权重，并生成基于证据的解释文本。

**输出格式（强制要求，直接供前端渲染）：**
- a. 证据清单（Evidence_List）：列出所有引用的指标值与新闻片段。
- b. 最终结论：方向与百分比区间（前端接管并换算为元/吨）。
- c. 多空雷达条形图：展示归一化后的量化基本面权重 vs 突发事件权重。
- d. 证据驱动推演正文：只允许"引用证据 + 解释"，绝对禁止脱离 Evidence_List 凭空拓展。
- e. 操作建议：仅限采购节奏建议。

## 五、前端功能与 SaaS 鉴权需求（UI/UX）

### 1. 账户系统与增值鉴权逻辑（Paywall）

- **鉴权策略：** 系统无免费自选池。后台管理员根据客户签单合同，写入其绑定的 SKU_ID 权限列表。用户登录后强制根据权限渲染。
- **增值引导（Upsell Hook）：** 导航栏显著位置设置带有"🔒锁图标"的【增加更多关注 SKU】按钮。点击后展示全量 SKU 灰态列表，点击未解锁 SKU 弹出提示："该标杆煤种暂未解锁，请联系您的专属客户经理开通预测权限"，并附带留资入口。

### 2. 行情预警雷达看板（Dashboard 主页）

- 根据用户权限动态渲染已解锁的 SKU 卡片。
- **UI 规范要求：** 必须遵循国内大宗交易习惯，**红色粗体↑ 代表上涨/利多**，**绿色粗体↓ 代表下跌/利空**。
- 展示：当前基准现货价（P0）、未来一旬预测方向、换算后的涨跌幅度区间（元/吨）及百分比（%）。
- 提供【查看本旬深度 AI 战报】按钮。

### 3. AI 证据链推演战报页（Drawer/Modal 详情页）

**版式结构：**
- **核心提要：** 预测结论。
- **多空因子博弈图：** 基于 Step3 的归一化权重，使用 ECharts 渲染横向条形图。
- **证据与逻辑正文：** 清晰分段展示结构化的文本分析报告（必须挂载引用的证据来源）。
- **强制免责声明：** 底部前端写死淡色文本："本报告基于 AI 生成，不构成配煤技术指导，请结合实际炉况谨慎决策。不对交易盈亏负责。"

## 六、容灾降级与风控底线（SLA）

### 1. 失败降级与熔断策略（SLA Downgrade）

- **爬虫节点宕机：** 若 Tool_B 抓取失败，主 Agent 平滑降级，仅基于 Tool_A 生成"纯基本面战报"，前端顶部增加黄条警示："今日暂无突发事件影响评估"。
- **量化 API 宕机：** 使用 T-1 日旧数据跑批，报告显著位置标记："基线数据存在延迟"。
- **大模型 API 超时（重试 3 次失败）：** 前端直接展示 Tool_A 的红绿箭头和涨跌幅区间，隐藏战报正文模块。

### 2. 大模型风控与"防幻觉"策略

**System Prompt 硬性约束（后端必须配置）：**
- "你是一名严谨的数据分析师，你的推演正文必须 100% 基于传入的 Evidence_List，严禁编造任何上下文未出现的数据、新闻或化验指标。"
- "严禁输出任何包含'配比、比例、增加 xx% 高硫'等干预生产工艺的关键词。若触发此类需求，必须替换为对采购节点（提前/延后买入）的建议。"
